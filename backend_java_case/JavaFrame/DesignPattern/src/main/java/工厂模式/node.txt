简单工厂其实不是一个设计模式,反而比较像是一种编程习惯
所有的工厂模式都用来封装对象的创建,工厂方法模式通过让子类决定改创建的对象是什么,来达到将对象创建的过程封装的目的
依赖倒置原则：
 代码里减少对具体类的依赖,要依赖抽象不要依赖具体类:不能让高层组件依赖低层组件,而且不管高层或低层组件,两者都应该依赖于抽象
 倒置指的是和一般OO设计思考方式完全相反:
   例如 比萨店进行准备:烘烤,装盒。。。一般思考先从顶端开始,然后往下倒具体类,正如你所看到的你不想让比萨店理会这些具体类,要不比萨店全部会依赖这些类,所以先从比萨开始思考能抽象什么?
下面的指导方针能帮你避免OO设计中违反依赖倒置原则:
  1.变量不可以持有具体类的引用:不去new
  2.不要让类派生自具体类,不要让类派生自具体类,请派生一个抽象类
  3.不要覆盖基类中已实现的方法
